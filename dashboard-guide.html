n8n x Vibe Coding x Claude Code

실시간 대시보드 구축 가이드 — n8n이 데이터를 수집하고, Claude Code가 Next.js 대시보드를 생성하고, Vercel로 배포합니다.

코드 한 줄 안 짜도 됩니다. 말로 설명하면 Claude Code가 다 만들어줍니다.

Vibe Coding

Vibe Coding = AI에게 말로 지시하는 개발 방식

코드를 직접 작성하는 대신, 원하는 결과를 자연어로 설명하면 AI가 코드를 생성합니다. 2025년 Andrej Karpathy가 처음 명명한 개념으로, 개발의 진입 장벽을 획기적으로 낮춥니다.

핵심은 "무엇을 만들지"에 집중하고, "어떻게 코딩할지"는 AI에게 맡기는 것입니다.

n8n Core Concepts

n8n = Node-based Workflow Automation

n8n은 노드(Node)를 연결하여 자동화 워크플로우를 만드는 오픈소스 도구입니다. 각 노드는 하나의 작업을 담당하며, 이들을 연결하면 복잡한 자동화가 완성됩니다.

핵심 노드 타입

T

Trigger Node

워크플로우를 시작하는 노드. Webhook, Cron, 이벤트 감지 등

A

Action Node

실제 작업을 수행. API 호출, DB 쿼리, 이메일 전송 등

F

Function Node

JavaScript로 데이터를 가공. JSON 변환, 필터링, 계산

C

Condition Node

조건 분기. IF/Switch로 데이터 흐름을 제어

H

HTTP Request

외부 API와 통신. GET, POST, PUT, DELETE 요청

R

Respond Node

Webhook 응답 반환. API 엔드포인트의 응답 데이터 설정

n8n 워크플로우 실행 흐름

API & Webhook Concepts

API = Application Programming Interface

서로 다른 소프트웨어가 데이터를 주고받는 규격화된 통신 방법입니다. 레스토랑에 비유하면, API는 주문서(메뉴)이고 데이터는 음식입니다. 정해진 형식(메뉴)으로 요청하면, 정해진 형식으로 응답이 옵니다.

Docker Concepts

Docker = Application Containerization

Docker는 애플리케이션과 그 실행 환경을 하나의 "컨테이너"로 패키징하는 기술입니다. 어떤 컴퓨터에서든 동일하게 실행됩니다. 배송에 비유하면, Docker 컨테이너는 규격화된 택배 상자이고, 안에 무엇이 들어있든 어디로든 동일하게 배달됩니다.

C

Image의 실행 인스턴스. 독립된 환경에서 실행되는 프로세스

Docker 기본 명령어

bash

docker pull nginx              # Docker Hub에서 이미지 받기

# Container 실행
docker run -d -p 80:80 nginx   # 백그라운드 실행, 포트 연결

# 실행 중인 컨테이너 확인
docker ps                      # 현재 실행 중인 목록

# 컨테이너 중지/삭제
docker stop [container_id]     # 중지
docker rm [container_id]       # 삭제

# Docker Compose (여러 컨테이너 한번에)
docker compose up -d           # 모든 서비스 시작
docker compose down            # 모든 서비스 중지
docker compose logs -f         # 실시간 로그 확인

로컬이 아닌 외부에서 접속하려면 도메인 + HTTPS 설정이 필요합니다. 두 가지 방법 중 선택하세요.

Method A: Caddy (초간단)

설정 2줄이면 HTTPS 자동 적용

난이도: 매우 낮음

Method B: Nginx + Certbot

장점: 풍부한 레퍼런스

난이도: 중간

Method A: Caddy (추천 — 가장 간단)

Caddy는 Let's Encrypt SSL 인증서를 자동으로 발급하고 갱신합니다. docker-compose.yml에 Caddy 서비스를 추가합니다:

Method B: Nginx + Let's Encrypt

nginx.conf
server {
    listen 80;
    server_name n8n.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name n8n.yourdomain.com;

    ssl_certificate     /etc/letsencrypt/live/n8n.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/n8n.yourdomain.com/privkey.pem;

    location / {
        proxy_pass http://n8n:5678;
        proxy_http_version 1.1;

        # WebSocket 지원 (n8n에 필수)
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # 프록시 헤더
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_buffering off;
    }
}

Core: Real-time Data Hook

hooks/useDashboardData.ts
import { useState, useEffect } from 'react';

export function useDashboardData(refreshInterval = 30000) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  const [lastUpdated, setLastUpdated] = useState(null);

  const fetchData = async () => {
    try {
      const res = await fetch('/api/data');
      if (!res.ok) throw new Error('데이터 로드 실패');
      const json = await res.json();
      setData(json);
      setLastUpdated(new Date());
      setError(null);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchData();
    const interval = setInterval(fetchData, refreshInterval);
    return () => clearInterval(interval);
  }, [refreshInterval]);

  return { data, loading, error, lastUpdated, refresh: fetchData };
}

n8n + Vercel — Realtime Integration

Method A: Polling

Dashboard polls n8n every 30 seconds

Pros: Simple, stable

Cons: Up to 30s delay

Method B: Push

n8n POSTs to Vercel on data update

Pros: True realtime

Cons: More setup required

Method B: Push Implementation

n8n 워크플로우 마지막에 HTTP Request 노드를 추가하여 Vercel API Route로 POST합니다.

app/api/webhook/route.ts
import { NextRequest, NextResponse } from 'next/server';

// Simple in-memory cache (use Redis/KV for production)
let cache: any = null;
let cacheTime: number = 0;

export async function POST(req: NextRequest) {
  const secret = req.headers.get('x-webhook-secret');
  if (secret !== process.env.WEBHOOK_SECRET) {
    return NextResponse.json(
      { error: 'Unauthorized' }, { status: 401 }
    );
  }

  cache = await req.json();
  cacheTime = Date.now();

  return NextResponse.json({ ok: true });
}

export async function GET() {
  return NextResponse.json(cache || { empty: true });
}
Core Insight — Method B는 n8n에서 Vercel 방향으로 데이터를 "Push"하는 방식입니다. n8n이 데이터를 수집한 즉시 대시보드에 반영되므로 진짜 실시간입니다.
