n8n 호스팅 — Self-Hosting vs Cloud

n8n을 사용하려면 먼저 n8n 인스턴스가 필요합니다

n8n 워크플로우를 실행하려면 n8n 서버가 어딘가에서 돌아가야 합니다. 크게 두 가지 방법이 있습니다: 셀프 호스팅(내 서버에 직접 설치)과 n8n Cloud(공식 SaaS 서비스)입니다.

3. 30초 만에 n8n 띄우기 (빠른 테스트)

먼저 n8n이 잘 돌아가는지 빠르게 확인합니다. 아래 한 줄이면 됩니다.

bash
# 한 줄로 n8n 실행 (테스트용)
docker run -it --rm --name n8n -p 5678:5678 n8nio/n8n

# 브라우저에서 열기:
# http://localhost:5678

# 종료: Ctrl + C (컨테이너 자동 삭제)
Tip — 위 명령은 테스트 전용입니다. --rm 옵션 때문에 컨테이너 종료 시 모든 데이터가 삭제됩니다. 실제 사용에는 아래의 Docker Compose 방식을 사용하세요.

4. Docker Compose로 프로덕션 설정

프로젝트 폴더를 만들고, docker-compose.yml과 .env 파일을 작성합니다.

bash — 프로젝트 디렉토리 생성
mkdir n8n-self-hosted
cd n8n-self-hosted
docker-compose.yml
version: '3.8'

services:
  n8n:
    image: n8nio/n8n:latest
    container_name: n8n
    restart: always
    ports:
      - "5678:5678"
    environment:
      # 기본 설정
      - GENERIC_TIMEZONE=Asia/Seoul
      - TZ=Asia/Seoul

      # Webhook URL (외부 접속 시 도메인으로 변경)
      - WEBHOOK_URL=http://localhost:5678/

      # 데이터베이스 (PostgreSQL 사용 시)
      - DB_TYPE=postgresdb
      - DB_POSTGRESDB_HOST=postgres
      - DB_POSTGRESDB_PORT=5432
      - DB_POSTGRESDB_DATABASE=n8n
      - DB_POSTGRESDB_USER=n8n
      - DB_POSTGRESDB_PASSWORD=${DB_PASSWORD}
    volumes:
      - n8n_data:/home/node/.n8n
    depends_on:
      postgres:
        condition: service_healthy
    networks:
      - n8n_network

  postgres:
    image: postgres:16-alpine
    container_name: n8n_postgres
    restart: always
    environment:
      - POSTGRES_DB=n8n
      - POSTGRES_USER=n8n
      - POSTGRES_PASSWORD=${DB_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - n8n_network
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U n8n"]
      interval: 5s
      timeout: 5s
      retries: 10

volumes:
  n8n_data:
  postgres_data:

networks:
  n8n_network:
    driver: bridge
.env — 환경변수 파일 (같은 디렉토리)
# PostgreSQL 비밀번호 (변경 필수!)
DB_PASSWORD=your_secure_password_here
볼륨의 중요성 — n8n_data 볼륨에 워크플로우, 자격증명, 실행 기록이 모두 저장됩니다. 볼륨이 없으면 컨테이너 재시작 시 모든 데이터가 사라집니다. 절대 볼륨 설정을 빼지 마세요.

6. 외부에서 접속하기 — 리버스 프록시 + HTTPS

로컬이 아닌 외부에서 접속하려면 도메인 + HTTPS 설정이 필요합니다. 두 가지 방법 중 선택하세요.

Method A: Caddy (초간단)

설정 2줄이면 HTTPS 자동 적용

장점: 가장 쉬움, SSL 자동

난이도: 매우 낮음

Method B: Nginx + Certbot

널리 쓰이는 웹서버, 세밀한 설정 가능

장점: 풍부한 레퍼런스

난이도: 중간

Method A: Caddy (추천 — 가장 간단)

Caddyfile
n8n.yourdomain.com {
    reverse_proxy n8n:5678
}

Caddy는 Let's Encrypt SSL 인증서를 자동으로 발급하고 갱신합니다. docker-compose.yml에 Caddy 서비스를 추가합니다:

docker-compose.yml — Caddy 서비스 추가
  caddy:
    image: caddy:2-alpine
    container_name: n8n_caddy
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
      - caddy_config:/config
    networks:
      - n8n_network

# volumes 섹션에 추가:
  caddy_data:
  caddy_config:

Method B: Nginx + Let's Encrypt

nginx.conf
server {
    listen 80;
    server_name n8n.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name n8n.yourdomain.com;

    ssl_certificate     /etc/letsencrypt/live/n8n.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/n8n.yourdomain.com/privkey.pem;

    location / {
        proxy_pass http://n8n:5678;
        proxy_http_version 1.1;

        # WebSocket 지원 (n8n에 필수)
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # 프록시 헤더
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_buffering off;
    }
}
bash — Certbot으로 SSL 인증서 발급
# Certbot 설치 (Ubuntu)
sudo apt-get install certbot python3-certbot-nginx

# SSL 인증서 발급 (자동으로 nginx.conf 수정)
sudo certbot --nginx -d n8n.yourdomain.com

# 자동 갱신 타이머 활성화
sudo systemctl enable certbot.timer
WEBHOOK_URL 변경 필수! — 외부 도메인 설정 후 .env 또는 docker-compose.yml의 WEBHOOK_URL을 https://n8n.yourdomain.com/으로 변경해야 합니다. 그래야 외부에서 Webhook이 정상 동작합니다.

# 5단계: 브라우저에서 정상 동작 확인

1-2. Creating API Endpoint with n8n Webhook

n8n에서 Webhook 노드를 추가하고, HTTP Method를 GET, Path를 /dashboard-data로 설정합니다. 마지막에 Respond to Webhook 노드를 연결합니다.

n8n Function Node — Data Format
// n8n Function 노드 코드
const rawData = $input.all();

return [{
  json: {
    summary: {
      totalCount: rawData.length,
      lastUpdated: new Date().toISOString(),
    },
    // 차트용 시계열 데이터
    timeSeries: rawData.map(item => ({
      date: item.json.date,
      value: item.json.count,
      label: item.json.category
    })),
    // 테이블용 상세 데이터
    tableData: rawData.map(item => ({
      id: item.json.id,
      title: item.json.title,
      status: item.json.status,
      timestamp: item.json.timestamp
    }))
  }
}];
Core Insight — n8n의 Webhook 노드 하나가 곧 REST API입니다. 별도 백엔드 서버 없이 n8n 자체가 API 서버 역할을 합니다.

Core: Real-time Data Hook

hooks/useDashboardData.ts
import { useState, useEffect } from 'react';

export function useDashboardData(refreshInterval = 30000) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  const [lastUpdated, setLastUpdated] = useState(null);

  const fetchData = async () => {
    try {
      const res = await fetch('/api/data');
      if (!res.ok) throw new Error('데이터 로드 실패');
      const json = await res.json();
      setData(json);
      setLastUpdated(new Date());
      setError(null);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchData();
    const interval = setInterval(fetchData, refreshInterval);
    return () => clearInterval(interval);
  }, [refreshInterval]);

  return { data, loading, error, lastUpdated, refresh: fetchData };
}

n8n + Vercel — Realtime Integration

Method A: Polling

Dashboard polls n8n every 30 seconds

Pros: Simple, stable

Cons: Up to 30s delay

Method B: Push

n8n POSTs to Vercel on data update

Pros: True realtime

Cons: More setup required

Method B: Push Implementation

n8n 워크플로우 마지막에 HTTP Request 노드를 추가하여 Vercel API Route로 POST합니다.

app/api/webhook/route.ts
import { NextRequest, NextResponse } from 'next/server';

// Simple in-memory cache (use Redis/KV for production)
let cache: any = null;
let cacheTime: number = 0;

export async function POST(req: NextRequest) {
  const secret = req.headers.get('x-webhook-secret');
  if (secret !== process.env.WEBHOOK_SECRET) {
    return NextResponse.json(
      { error: 'Unauthorized' }, { status: 401 }
    );
  }

  cache = await req.json();
  cacheTime = Date.now();

  return NextResponse.json({ ok: true });
}

export async function GET() {
  return NextResponse.json(cache || { empty: true });
}
Core Insight — Method B는 n8n에서 Vercel 방향으로 데이터를 "Push"하는 방식입니다. n8n이 데이터를 수집한 즉시 대시보드에 반영되므로 진짜 실시간입니다.

Step 1. Google Drive 설정

영수증 저장 폴더 생성

Google Drive에 영수증_OCR 폴더를 만들고, 공유 설정을 Anyone with link → Viewer로 변경합니다.

URL 형식 — Google Drive는 다음 URL로 이미지에 직접 접근할 수 있습니다. Google 공식 문서에서도 uc?id= 방식이 직접 다운로드 링크로 사용됩니다.
Google Drive Direct URL Format
// 올바른 URL 형식
https://drive.google.com/uc?export=view&id=FILE_ID

// 잘못된 URL 형식 (OpenAI에서 접근 불가)
// https://drive.google.com/file/d/FILE_ID/view

Step 2. n8n에서 이미지 URL 생성

Google Drive 노드에서 File ID를 얻고, Public URL을 조합합니다.

n8n Expression
// File ID 참조
{{$json["id"]}}

// 이미지 직접 접근 URL 생성
https://drive.google.com/uc?export=view&id={{$json["id"]}}
